### B.词语意义的了解
##### 1.Repository：仓库（用于存储项目文件（包括代码、文档、图像等各种资源）的地方。可以把它想象成一个装满与特定项目相关物品的箱子。例如，如果你正在开发一个网站，那么这个网站的 HTML 文件、CSS 样式表、JavaScript 脚本以及相关的设计文档都可以存放在同一个仓库中）
##### 2.Star:可以粗略理解为点赞加收藏，亦可理解为关注，“Star” 数量在一定程度上反映了它的受欢迎程度和质量，“Star”数越多，仓库的资源和质量越有价值
##### 3.Pull request：Pull Request（PR）就像是一个 “合并请求”。你在自己的代码小天地（分支）里捣鼓，比如修改了错误或者添加了新东西。弄完后，你跟仓库的 “管理员”（维护者）说：“我这儿有一些改动，你看看能不能把它们合并到主代码那边去。” 这个请求就是 Pull Request
##### 4.Fork：Fork 就像是在 GitHub 上 “复制” 一个仓库。你看到一个别人的代码仓库，觉得挺有意思，想自己动手改改或者研究研究，就可以使用 Fork 操作。这就好比你看到别人做了一个乐高模型（原仓库），你很喜欢，于是你就复制了一份乐高积木（Fork 后的仓库），这些积木现在就完全在你的手上，你可以按照自己的想法重新搭建（以上一些部分源于[豆包](https://www.doubao.com/chat/268571857841154)）
