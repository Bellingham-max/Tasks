## 4.git与github的关系：
### （1）Git 是分布式版本控制系统，用于管理文件版本变化。GitHub 是基于 Web 的 Git 仓库托管服务，提供远程存储及代码审查、问题跟踪、团队协作等附加功能。
### （2）Git 用于本地版本控制操作，GitHub 用于存储 Git 仓库，其协作功能也是基于 Git，二者紧密联系，方便代码的备份、共享和团队开发。
### （3）可形象理解为：
#### A.可以把 Git 想象成一个超级智能的文件管家。这个管家待在你的电脑里，它会仔细地记下你文件（特别是代码之类的）每次修改的小细节。比如说你在写一个程序，今天加了个新功能，明天又修改了个小错误，Git 这个管家就会把这些都记在小本本上，方便你以后查看。
#### b.GitHub 呢，就像是一个网上的大仓库。你可以把 Git 管家记下来的那些小本本都放到这个网上仓库里。这就好比你有一些很重要的笔记，你担心放在家里会丢，就把它们放在网上仓库里保存起来。而且啊，这个网上仓库还有很多方便的功能呢。你可以让你的小伙伴也能看到这些笔记，然后大家一起在上面写写画画，一起改进这个程序，这就起到了团队协作的作用。
#### c.总的来说，Git 是在你自己电脑里管文件变化的，GitHub 是把这些变化放到网上让人能看到，还能一起干活的地方。
